Harness Configuration

Configure core run inputs, then launch your existing suite. Advanced options are available when needed.

?Field help for Deployment ID

Required ID for your Genesys Web Messaging deployment. If this is wrong, the harness cannot start conversations.

?Field help for Region

Genesys Cloud region domain (for example mypurecloud.com or usw2.pure.cloud). It controls API and web origins used during the run.

?Field help for Judge Execution Mode

Select how Gemma 4 judge routing works for this run.

  • single: use one selected Gemma model for the run.
  • dual_strict_fallback: start on gemma4:e4b and escalate to gemma4:31b only when an evaluation result is unusable or low confidence.
?Field help for Single Judge Model

Gemma model used in single mode. In dual mode, the app always uses the fixed pair gemma4:e4b -> gemma4:31b.

Effective primary judge: {{ effective_harness_judge_model }}
?Field help for Max Conversation Turns

Maximum user turns allowed per attempt. Lower values finish faster, higher values allow deeper conversations but can increase run duration and timeout risk.

?Field help for Harness Mode

Select how attempts are judged for this run.

  • standard: Uses intent/goal validation flow.
  • journey: Uses contained + fulfilled path journey validation and ignores expected_intent.
  • Use Suite / Config Default: Keeps the mode defined in suite/config.
?Field help for Journey Category Strategy

Controls how journey scenarios resolve primary category when journey mode is active.

  • rules_first: More deterministic and stable across runs.
  • llm_first: More flexible interpretation, but can be less consistent.
  • Use Config Default: Uses the configured application default.
?Field help for Test Suite File

Upload the suite to execute for this run. Supported file types: .json, .yaml, .yml. Invalid schema content prevents run start.

Advanced Settings
?Field help for Custom Ollama Model Override

Advanced single-mode override for non-Gemma or custom Ollama models. Ignored when dual_strict_fallback is selected.

?Field help for Enable Parallel Attempt Workers

Runs attempt execution with a shared worker pool for faster suites. Disable to force serial mode for troubleshooting or controlled comparisons.

?Field help for Max Parallel Attempt Workers

Worker pool size when parallel execution is enabled. Valid range is 1..3 (default 2); higher values increase throughput but also raise API and CPU pressure. Attempt start pacing is globally rate-limited by min_attempt_interval_seconds.

?Field help for Knowledge Mode Timeout

Timeout floor applied only to knowledge-mode attempts. Effective response and step timeout become max(base timeout, knowledge timeout), so slower knowledge replies have more time without changing non-knowledge scenarios.

?Field help for Enable Phase 11 Judging Mechanics

When enabled, judge-driven outcomes use deterministic scoring thresholds in addition to hard gates. Default is OFF for safe rollout.

?Field help for Judging Objective Profile

Controls how score weighting is interpreted for judge-driven paths.

  • blended: balanced across category + journey checks.
  • intent_focused: prioritizes category/intent correctness.
  • journey_focused: prioritizes containment + fulfillment/path behavior.
?Field help for Judging Strictness

Threshold profile for passing score checks. strict is most conservative, lenient is most forgiving.

?Field help for Judging Tolerance

Subtracts from the strictness threshold (0.00 to 0.45). Useful for controlled relaxation without changing strictness profile.

?Field help for Journey Weight Containment

Weight applied to containment when scoring journey attempts. Weights are normalized automatically.

?Field help for Journey Weight Fulfillment

Weight applied to fulfillment (goal achieved) in journey score calculation.

?Field help for Journey Weight Path

Weight applied to path-correctness in journey score calculation.

?Field help for Judging Explanation Mode

Controls how much score detail is appended to Judge explanations.

  • concise: one-line score summary.
  • standard: score + threshold + profile.
  • verbose: includes criterion breakdown values.
?Field help for Enable Phase 12 Journey Dashboard

Shows dynamic journey taxonomy views in Results and includes journey rollups in exports. Default is OFF for gradual rollout.

?Field help for Genesys OAuth Client ID

OAuth client ID used for Conversations API calls (intent fallback and transcript import workflows). Leave blank if your runs do not require those API paths.

?Field help for Genesys OAuth Client Secret

OAuth secret paired with the client ID. Required when fallback intent resolution or transcript import requires authenticated Genesys API access.

?Field help for Intent Participant Attribute Name

Participant data attribute read during intent fallback. Use this when your bot writes intent to a custom attribute key instead of detected_intent.

?Field help for Capture Debug WebSocket Frames

Collects compact frame metadata on each attempt. Useful for debugging missing conversation_id/participant_id, with a small runtime overhead.

?Field help for Debug Frame Limit

Maximum number of debug frame summaries stored per attempt. Higher limits provide deeper diagnostics but increase attempt payload size.
